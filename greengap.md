# GGF — Green Gap Finder

> 서울시 녹지 부족 구역 탐지 및 친환경 도시 설계 의사결정 지원 시스템

---

## 프로젝트 개요

**GGF(Green Gap Finder)** 는 서울시를 대상으로 토지피복도, 공원, 인구 데이터를 통합하여 **녹지 부족 구역을 정량적으로 탐지**하고, 2040 서울도시기본계획의 공간·환경 목표에 부합하는 **친환경 도시 설계 대안을 우선순위 형태로 제안**하는 데이터 기반 의사결정 지원 시스템입니다.

본 시스템은 단순한 공간 분석을 넘어, 다양한 공공 데이터를 자동으로 처리하는 **GIS 기반 데이터 파이프라인과 웹 서비스 구조를 결합한 플랫폼**으로 설계됩니다.

---

## 문제 정의 및 필요성

현대 도시의 급격한 시가화는 인구 증가와 건축물 밀집, 교통량 증가를 동반하며 다음과 같은 복합적 환경 문제를 심화시키고 있습니다.

-  **도시 열섬 현상** — 불투수면 증가로 인한 야간 냉각 저하
-  **미세먼지 축적** — 교통량 및 산업 활동 증가로 인한 대기질 악화
-  **녹지 공간 감소** — 일부 지역 1인당 공원 면적이 법적 권장 기준 미달

서울시는 전국 국토 면적의 약 **0.6%** 에 전국 인구의 약 **20%** 가 집중된 초밀집도시로, 이러한 문제가 특히 두드러집니다.

> **2040 서울도시기본계획** 의 주요 목표:
> - 보행 10분(약 800m) 이내 공원 접근권 확보
> - 2040년까지 공원·녹지율 약 **31%** 달성
> - 도시 녹지 네트워크 강화 및 열환경 개선

## ✨ 핵심 특징

### 1. 정책 정합성
- **최상위 법정 계획 연계**: 2040 서울도시기본계획의 녹지·공원 확충 방향을 분석 모델의 기본 제약 조건으로 설정
- **공원·녹지 정책 흐름 반영**: 단순 녹지 면적이 아닌 접근성과 연결성을 고려한 분석 구조 적용
- **정량 목표 기반 평가**: 지역별 녹지 수준을 정량적으로 평가하고 정책 목표 대비 달성 수준 비교

### 2. 자동화 파이프라인
- **현황 진단**: 녹지 면적, 1인당 녹지 면적, 공원 접근성 등 지표 자동 산출 및 지도 시각화
- **취약도 보정**: 대기질, 지표면 온도 등 환경 취약 지표 결합으로 개입 필요성 정량화
- **정책 기반 후보지 도출**: 형평성·환경 개선 효과·접근성·실행 가능성을 종합 점수화

| 친환경 도시 설계 지원 모델 | 법적 기준 및 2040 계획 방향성을 반영한 도시 설계 추천 모델 개발 |

- **정책 및 설계 의사결정 지원**: 녹지 부족 지역 식별 및 등급화, 예산 분배·사업 계획 수립의 데이터 기반 근거 제공

## 🔄 데이터 파이프라인

```
Raw Data → Data Processing → Feature Engineering → Spatial Aggregation → Green Gap Analysis → Output
```

### 단계별 설명

#### 1️⃣ Raw Data (데이터 수집)
- 토지피복도 (환경부)
- 공원 데이터
- 인구 데이터
- 행정동 경계 데이터

#### 2️⃣ Data Processing (전처리)
- 좌표계 통일
- 데이터 정제
- 타일 데이터 병합(Merge)
- 녹지 필터링 (산림, 초지 등 추출)

#### 3️⃣ Feature Engineering (특징 생성)
- 녹지 면적 계산
- 녹지 비율 계산
- 공원 접근성 계산 (거리 연산)
- 1인당 녹지 면적 도출

#### 4️⃣ Spatial Aggregation (공간 집계)
- 행정동 단위 집계
- 지표 통합 (녹지 총량, 평균 접근성, 인구 대비 녹지 수준)

#### 5️⃣ Green Gap Analysis (녹지 부족 지역 분석)
- 정책 기준 기반 판단
- 녹지 부족 지역 라벨링

#### 6️⃣ Output Data
- `GeoJSON` — 공간 분석 결과
- `CSV` — 통계 데이터

#### 7️⃣ Web Service
- **API 서버**: FastAPI
- **지도 시각화**: Leaflet
- **사용자 인터페이스**

## 기능 발전 로드맵

### Phase 1 — 규칙 기반 자동화 파이프라인 구성
> Python 기반 비 머신러닝 녹지 진단 체계 구현

- 토지피복도 재분류 (녹지/비녹지)
- 1인당 녹지 면적, 생활권 공원 접근성, 녹지 비율 등 지표 산출
- 전처리 → 녹지 추출 → 공간 단위 집계 → 지표 계산 전 과정 자동화
- 이후 ML 모델 학습을 위한 라벨 데이터셋 생성

### Phase 2 — 머신러닝 기반 미시 녹지 탐지 모듈 구축
> 딥러닝 세그멘테이션 모델(U-Net 등)을 활용한 고해상도 분석

- Phase 1 라벨 데이터 기반 딥러닝 모델 학습
- 위성/항공 영상에서 픽셀 단위 녹지 자동 분류
- 가로수, 소규모 공원, 옥상 녹화 등 세밀한 녹지 요소 탐지
- 자동화 파이프라인 통합 (데이터 입력 → 학습 → 저장 → 재사용)

### Phase 3 — 웹 기반 녹지 분석 및 의사결정 지원 서비스 구현
> 시민 및 행정기관이 활용 가능한 웹 서비스 배포

- FastAPI 백엔드를 통한 분석 결과 API 제공
- Leaflet 기반 프론트엔드 지도 시각화
- 지역 선택 시 녹지율, 1인당 녹지 면적, 공원 접근성 실시간 조회
- 새로운 데이터 입력 시 분석 결과 자동 갱신
